# Быстрое решение проблемы с Git и Amvera

Если в Amvera нет опции создания токена доступа, используйте один из этих способов.

## Решение 1: Использование SSH ключей (рекомендуется)

### Шаг 1: Проверьте, есть ли у вас SSH ключ

Откройте PowerShell или Git Bash и выполните:

```powershell
# Проверить существующие ключи
ls ~/.ssh
```

Если видите файлы `id_rsa.pub` или `id_ed25519.pub` - ключ уже есть.

### Шаг 2: Создайте SSH ключ (если его нет)

```powershell
# Создать новый SSH ключ
ssh-keygen -t ed25519 -C "your_email@example.com"
```

**Важно:**
- Нажмите Enter для всех вопросов
- Или укажите пароль для защиты ключа (рекомендуется)
- Файл будет создан в `C:\Users\Ivan\.ssh\id_ed25519`

### Шаг 3: Скопируйте публичный ключ

```powershell
# Показать публичный ключ
Get-Content ~/.ssh/id_ed25519.pub
```

Или если используете RSA:
```powershell
Get-Content ~/.ssh/id_rsa.pub
```

**Скопируйте весь вывод** (начинается с `ssh-ed25519` или `ssh-rsa` и заканчивается вашим email).

### Шаг 4: Добавьте ключ в Amvera

1. Войдите в личный кабинет Amvera: https://amvera.ru
2. Найдите раздел "Настройки" или "Профиль"
3. Ищите раздел:
   - "SSH ключи"
   - "SSH Keys" 
   - "Ключи доступа"
   - "Настройки безопасности"
4. Нажмите "Добавить SSH ключ" или "Add SSH Key"
5. Вставьте скопированный публичный ключ
6. Сохраните

**Если не можете найти раздел SSH:**
- Попробуйте поискать в меню профиля (правый верхний угол)
- Или в разделе "Безопасность" / "Security"
- Или обратитесь в поддержку Amvera

### Шаг 5: Клонируйте через SSH

```bash
git clone git@git.msk0.amvera.ru:overmotivated/astrolhub.git
```

## Решение 2: Использование пароля от аккаунта

Если Amvera позволяет использовать пароль от аккаунта:

### Вариант A: Интерактивный ввод

```bash
git clone https://git.msk0.amvera.ru/overmotivated/astrolhub.git
```

Когда запросит:
- **Username**: `overmotivated` (или ваш логин Amvera)
- **Password**: ваш пароль от аккаунта Amvera

### Вариант B: В URL (небезопасно, но работает)

```bash
git clone https://overmotivated:YOUR_PASSWORD@git.msk0.amvera.ru/overmotivated/astrolhub.git
```

**⚠️ ВНИМАНИЕ**: Пароль будет виден в истории команд! Используйте только для теста.

---

## Решение 3: Клонирование через веб-интерфейс Amvera

1. Войдите в личный кабинет Amvera
2. Найдите ваш проект `astrolhub`
3. Откройте страницу проекта
4. Ищите кнопку "Клонировать" или "Clone"
5. Скопируйте URL (может быть уже с учетными данными)
6. Используйте этот URL для клонирования

---

## Решение 4: Настройка через Git Credential Manager

### Windows Credential Manager

1. Попробуйте клонировать обычным способом:
   ```bash
   git clone https://git.msk0.amvera.ru/overmotivated/astrolhub.git
   ```

2. Когда Windows запросит учетные данные:
   - Введите логин Amvera
   - Введите пароль от аккаунта
   - Отметьте "Запомнить"

3. Windows сохранит учетные данные в Credential Manager

### Настройка Git для сохранения паролей

```bash
# Включить сохранение учетных данных
git config --global credential.helper manager-core

# Или для Windows
git config --global credential.helper wincred
```

## Решение 5: Использование GitHub вместо Git Amvera

Если у вас есть доступ к GitHub:

1. Создайте репозиторий на GitHub
2. Загрузите туда код
3. В Amvera подключите GitHub репозиторий вместо Git Amvera

Это может быть проще, так как GitHub имеет более понятный интерфейс для токенов.

---

## Проверка подключения SSH

После настройки SSH ключа проверьте:

```bash
ssh -T git@git.msk0.amvera.ru
```

Должно вернуть сообщение об успешной аутентификации.

Если видите ошибку "Permission denied":
1. Убедитесь, что ключ добавлен в Amvera
2. Проверьте правильность URL (должен быть `git@git.msk0.amvera.ru`, не `https://`)

## Альтернатива: Работа через веб-интерфейс

Если Git не работает, можно:

1. Загрузить код через веб-интерфейс Amvera (если есть такая опция)
2. Использовать GitHub и подключить его к Amvera
3. Обратиться в поддержку Amvera для настройки доступа

## Быстрая команда для проверки

Попробуйте выполнить эту команду и посмотрите, что запросит:

```bash
git ls-remote https://git.msk0.amvera.ru/overmotivated/astrolhub.git
```

Это покажет, какие учетные данные нужны.
