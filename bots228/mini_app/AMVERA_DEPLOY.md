# Деплой на Amvera - Подробная инструкция

Пошаговое руководство по развертыванию mini_app на платформе Amvera.

## 📋 Предварительные требования

1. ✅ Аккаунт на Amvera (https://amvera.ru)
2. ✅ Репозиторий на GitHub или Git репозиторий Amvera
3. ✅ Код проекта загружен в репозиторий

**Примечание**: Если вы используете Git репозиторий Amvera, см. [AMVERA_GIT_SETUP.md](AMVERA_GIT_SETUP.md) для настройки аутентификации.

## 🚀 Шаг 1: Подготовка репозитория

### 1.1 Убедитесь, что все файлы на месте

В репозитории должны быть:
- ✅ `main.py` - главный файл приложения
- ✅ `requirements.txt` - зависимости Python
- ✅ `amvera.yml` - конфигурация для Amvera (опционально)
- ✅ `Dockerfile` - для Docker деплоя (опционально)
- ✅ `.gitignore` - исключает `.env` и другие файлы

### 1.2 Проверьте .gitignore

Убедитесь, что `.env` файл в `.gitignore`:
```gitignore
.env
*.db
```

## 🔧 Шаг 2: Создание проекта в Amvera

### 2.1 Вход в систему

1. Перейдите на https://amvera.ru
2. Войдите в личный кабинет
3. Нажмите "Создать проект"

### 2.2 Подключение репозитория

1. Выберите "Подключить репозиторий GitHub"
2. Авторизуйтесь через GitHub (если нужно)
3. Выберите ваш репозиторий
4. Выберите ветку `main` или `master`

### 2.3 Настройка проекта

**Основные настройки:**
- **Имя проекта**: `mini-app` (или любое другое)
- **Тип приложения**: 
  - Если есть `Dockerfile` → выберите "Docker"
  - Если есть `amvera.yml` → выберите "Python"
  - Иначе → выберите "Python" и укажите команду запуска вручную

**Корневая директория:**
- Если репозиторий содержит только `mini_app/` → укажите `mini_app`
- Если весь проект в корне → оставьте пустым или укажите `.`

## 🗄️ Шаг 3: Настройка базы данных PostgreSQL

### 3.1 Добавление PostgreSQL

1. В настройках проекта перейдите в "База данных"
2. Нажмите "Добавить базу данных"
3. Выберите "PostgreSQL"
4. Выберите тариф:
   - **Бесплатный** - для тестирования (ограниченные ресурсы)
   - **Платный** - для продакшена

### 3.2 Получение DATABASE_URL

После создания базы данных:
1. Amvera автоматически создаст переменную `DATABASE_URL`
2. Она будет доступна в разделе "Переменные окружения"
3. Формат: `postgresql://user:password@host:port/database`

## 🔐 Шаг 4: Настройка переменных окружения

В разделе "Переменные окружения" добавьте:

### Обязательные переменные:

```env
DATABASE_URL=postgresql://...  # Автоматически создается при добавлении PostgreSQL
OPENROUTER_API_KEY=sk-or-v1-...  # Ваш API ключ от OpenRouter
```

### Как добавить переменные:

1. Перейдите в "Настройки" → "Переменные окружения"
2. Нажмите "Добавить переменную"
3. Введите имя: `OPENROUTER_API_KEY`
4. Введите значение: ваш API ключ
5. Нажмите "Сохранить"

**Важно**: 
- ❌ НЕ добавляйте `.env` файл в репозиторий
- ✅ Используйте только переменные окружения в интерфейсе Amvera
- ✅ Переменные доступны только вашему приложению

## ⚙️ Шаг 5: Настройка команды запуска

### Вариант 1: Через amvera.yml (рекомендуется)

Если файл `amvera.yml` присутствует в репозитории, Amvera автоматически использует его:

```yaml
start: uvicorn main:app --host 0.0.0.0 --port 8080
```

### Вариант 2: Через настройки в интерфейсе

1. Перейдите в "Настройки" → "Команда запуска"
2. Укажите:
   ```
   uvicorn main:app --host 0.0.0.0 --port 8080
   ```

### Вариант 3: Через Dockerfile

Если используете Docker:
1. Amvera автоматически обнаружит `Dockerfile`
2. Используйте команду из Dockerfile

## 🚀 Шаг 6: Первый деплой

### 6.1 Запуск сборки

1. Нажмите кнопку "Деплой" или "Собрать и запустить"
2. Amvera начнет процесс:
   - Клонирование кода из GitHub
   - Установка зависимостей
   - Сборка приложения
   - Запуск

### 6.2 Мониторинг процесса

В разделе "Логи" вы можете наблюдать:
- Установку зависимостей
- Ошибки сборки (если есть)
- Запуск приложения

**Время сборки**: обычно 2-5 минут

### 6.3 Проверка статуса

После завершения сборки:
- ✅ Зеленый индикатор = приложение запущено
- ❌ Красный индикатор = ошибка (проверьте логи)

## 🗄️ Шаг 7: Миграция базы данных

После успешного деплоя нужно создать таблицы в PostgreSQL:

### Способ 1: Через консоль Amvera

1. Перейдите в раздел "Консоль" или "SSH"
2. Откройте терминал
3. Выполните:
   ```bash
   python migrate_to_postgres.py
   ```

### Способ 2: Через веб-интерфейс

1. Раздел "Задачи" → "Выполнить команду"
2. Команда: `python migrate_to_postgres.py`
3. Нажмите "Выполнить"

### Способ 3: Локально с подключением к удаленной БД

1. Скопируйте `DATABASE_URL` из Amvera
2. Создайте локальный `.env` файл:
   ```env
   DATABASE_URL=postgresql://...из_amvera...
   ```
3. Запустите миграцию локально:
   ```bash
   python migrate_to_postgres.py
   ```

## ✅ Шаг 8: Проверка работы

### 8.1 Получение URL приложения

После деплоя вы получите URL вида:
```
https://your-app-name.amvera.io
```

### 8.2 Проверка endpoints

Откройте в браузере:
- `https://your-app-name.amvera.io/` - главная страница
- `https://your-app-name.amvera.io/numerology` - страница нумерологии
- `https://your-app-name.amvera.io/sonnik` - страница сонника

### 8.3 Проверка API

Используйте curl или Postman:
```bash
curl https://your-app-name.amvera.io/
```

## 🔄 Автоматический деплой

Amvera автоматически задеплоит приложение при каждом push в ветку `main`:

1. Сделайте изменения в коде
2. Закоммитьте и запушьте:
   ```bash
   git add .
   git commit -m "Update"
   git push origin main
   ```
3. Amvera автоматически начнет новый деплой

## 🌐 Шаг 9: Настройка домена (опционально)

### 9.1 Добавление собственного домена

1. В настройках проекта → "Домены"
2. Нажмите "Добавить домен"
3. Введите ваш домен (например, `app.yourdomain.com`)

### 9.2 Настройка DNS

Amvera предоставит инструкции по настройке DNS:
1. Добавьте CNAME запись в DNS вашего домена
2. Укажите значение, предоставленное Amvera
3. Подождите распространения DNS (до 24 часов)

### 9.3 SSL сертификат

Amvera автоматически выдаст SSL сертификат для вашего домена.

2. Выберите нужный период времени
3. Фильтруйте по уровню (INFO, ERROR, WARNING)

1. Раздел "Метрики"
2. Доступны:
   - Использование CPU

   - Количество запросов
   - Время отклика

### Настройка алертов

1. Раздел "Алерты"
2. Настройте уведомления при:
   - Ошибках в приложении
   - Превышении лимитов ресурсов
   - Недоступности приложения

## 🔧 Устранение проблем

### Проблема: Приложение не запускается

**Решение:**
1. Проверьте логи в разделе "Логи"
2. Убедитесь, что команда запуска правильная
3. Проверьте, что порт указан как `8080` или используется переменная `PORT`

### Проблема: Ошибка подключения к базе данных

**Решение:**
1. Проверьте, что PostgreSQL создан и запущен
2. Убедитесь, что `DATABASE_URL` правильно настроена
3. Проверьте, что миграция выполнена

### Проблема: Зависимости не устанавливаются

**Решение:**
1. Проверьте `requirements.txt` на наличие всех зависимостей
2. Убедитесь, что версии Python совместимы
3. Проверьте логи сборки на ошибки

### Проблема: Порт занят

**Решение:**
- Amvera автоматически управляет портами
- Используйте переменную `PORT` в коде
- Не указывайте фиксированный порт
